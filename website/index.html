Learning path

A shorter path from GitHub to a running program.

The project has one public compiler path and one public checker path. These are the commands to keep in front of new readers.

1

Clone and check the toolchain

git clone https://github.com/vaislang/vais
cd vais
scripts/vaisc doctor
2

Run a verified example

scripts/vaisc run examples/e90_enum_wildcard.vais
scripts/vais-check examples/e90_enum_wildcard.vais
3

Build native output

scripts/vaisc emit-ir examples/c4.vais -o /tmp/c4.ll
scripts/vaisc build examples/c4.vais -o /tmp/c4
/tmp/c4

Language

Current syntax is intentionally small and gate-backed.

The public surface uses full-word syntax: fn, let, if, while, for, struct, enum, match, and return. Older compact or Rust-like examples are stale for this mainline.

Scalars

Int, Bool, Char, and Str are verified across promoted paths where documented.

Collections

List, List, and local Map slices are verified. Broader generics, Option, and Result remain future work.

Enums and match

Payload-free enums, Int payload enums, wildcard match arms, and selected struct payload cases are protected by the release corpus.

Imports

The full engine supports local dotted imports and local dependency package paths through vais.toml.

Compiler

The public compiler is scripts/vaisc.

The native public driver links the reusable self-host compiler core, emits LLVM IR, and uses clang for native binaries. Standalone installs are built from this repository.

scripts/vaisc doctor
scripts/vaisc run examples/c4.vais
scripts/vaisc build examples/c4.vais -o /tmp/c4

scripts/install-vaisc.sh --prefix "$HOME/.local"
"$HOME/.local/bin/vaisc" doctor

Verification

Public claims come from gates, not promises.

The site and GitHub README should stay synced to these checks. If a feature is not in the gates or parity manifest, it should not be advertised as current Vais.
