Why Vais

Vais is being kept as one named language and one compiler workflow: source files use the .vais extension, and the public command is scripts/vaisc.

The useful boundary is simple: document only the syntax and compiler behavior protected by the repository gates, then expand the language by adding tests and self-host coverage first.

That keeps the project readable for humans, usable by tools, and honest about what the compiler can build today.
